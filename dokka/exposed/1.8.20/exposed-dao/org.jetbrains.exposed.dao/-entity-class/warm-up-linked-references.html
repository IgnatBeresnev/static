optimizedLoad

will force to make to two queries to load ids and referenced entities separately. Can be useful when references target the same entities. That will prevent from loading them multiple times (per each reference row) and will require less memory/bandwidth for "heavy" entities (with a lot of columns or columns with huge data in it)
