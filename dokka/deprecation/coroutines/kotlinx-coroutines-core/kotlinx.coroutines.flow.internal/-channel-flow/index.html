ChannelFlow

abstract class ChannelFlow<T>(    val context: CoroutineContext,     val capacity: Int,     val onBufferOverflow: BufferOverflow) : FusibleFlow<T> (source)

Operators that use channels as their "output" extend this ChannelFlow and are always fused with each other. This class servers as a skeleton implementation of FusibleFlow and provides other cross-cutting methods like ability to produceIn the corresponding flow, thus making it possible to directly use the backing channel if it exists (hence the ChannelFlow name).

This an internal API and should not be used from general code.**

open fun dropChannelOperators(): Flow<T>?

When this ChannelFlow implementation can work without a channel (supports Channel.OPTIONAL_CHANNEL), then it should return a non-null value from this function, so that a caller can use it without the effect of additional flowOn and buffer operators, by incorporating its context, capacity, and onBufferOverflow into its own implementation.

produceImpl
Link copied to clipboard
open fun produceImpl(scope: CoroutineScope): ReceiveChannel<T>

Here we use ATOMIC start for a reason (#1825). NB: produceImpl is used for flowOn. For non-atomic start it is possible to observe the situation, where the pipeline after the flowOn call successfully executes (mostly, its onCompletion) handlers, while the pipeline before does not, because it was cancelled during its dispatch. Thus onCompletion and finally blocks won't be executed and it may lead to a different kinds of memory leaks.
